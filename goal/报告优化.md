## 2. 核心设计原则

### 2.1 报告存储引用，报告界面渲染内容

报告 Artifact 可以继续保存结构化图表引用：

```markdown
<visualization
  artifactId="artifact_chart_001"
  title="五级分类贷款余额分布"
/>
```

但完整报告查看器必须将其解析为真实图表：

```text
Markdown visualization 节点
→ 解析 artifactId
→ 读取 Chart Artifact
→ 校验 VisualizationSpec
→ 调用现有 Chart Renderer
→ 在报告正文位置渲染真实图表
```

用户最终看到的是图表，不是引用标签。

### 2.2 不复制图表数据

不得在报告 Markdown 中写入：

* 完整 ECharts option；
* 完整 Recharts 配置；
* 图表数据 Base64；
* SVG 字符串；
* Canvas 快照；
* 大段 JSON 数据；
* 图片 Data URL。

图表数据和协议继续保存在 Chart Artifact 中。

### 2.3 Artifact 是数据血缘，不是用户展示内容

保留：

* `artifactId`；
* `visualizationArtifactIds`；
* 来源 SQL Artifact；
* 来源 Python Artifact；
* 图表版本；
* 数据血缘。

但不得在报告正文直接展示内部 Artifact ID。

### 2.4 单一图表渲染器

报告内图表应复用现有：

```text
VisualizationRenderer
ChartRenderer
VisualizationSpecRenderer
```

中的实际组件。

不要创建：

```text
ReportOnlyChartRenderer
```

之类的第二套平行实现，除非当前代码结构确实无法复用；即便如此，也应提取共享 Renderer，而不是复制代码。

## 15. Neutral 主题视觉要求

### 15.1 整体风格

图表应呈现：

* 中性；
* 克制；
* 清晰；
* 低装饰；
* 低饱和；
* 高可读性；
* 与 Astryx 页面背景和卡片一致。

避免：

* 彩虹配色；
* 高饱和蓝红绿；
* 大面积渐变；
* 发光效果；
* 3D 效果；
* 粗重阴影；
* 过度圆角；
* 过密网格线；
* 过重坐标轴。

### 15.2 背景

报告内图表默认使用：

```text
transparent
```

或与报告内容区域一致的 neutral surface。

不要在图表外再套一层突兀的白色卡片。

### 15.3 边框

如需要容器边界：

* 使用 neutral border；
* 细线；
* 不使用强阴影；
* 圆角与 Astryx 常规内容容器一致。

### 15.4 文字

* 标题使用 Astryx 正文字体；
* 图表标题 medium；
* 坐标轴、图例和辅助文字使用 muted foreground；
* 不自行加载其他字体；
* 中文和英文保持一致字体栈。

## 16. 系列颜色策略

### 16.1 单系列图表

单系列优先使用 Neutral Theme 的主 Accent 或第一语义序列色。

不得为不同分类随机生成颜色。

### 16.2 多系列图表

多系列使用受控序列色：

```ts
theme.series
```

要求：

* 低饱和；
* 深浅模式均可辨认；
* 相邻系列具有足够差异；
* 不依赖颜色作为唯一信息；
* 图例和 Tooltip 必须明确标识。

### 16.3 风险语义颜色

只有具备明确业务语义时才允许使用：

* 正常：neutral / positive；
* 关注：warning；
* 次级、可疑、损失：negative 的深浅变化。

不得将任意最大值自动标红。

如果当前图表协议没有声明语义类型，则使用 neutral 系列色，不自行猜测风险含义。

## 17. 各图表类型主题规范

### 17.1 柱状图

### 17.2 横向柱状图

要求：

* 左侧留出足够空间；
* 不因长中文字段截断核心含义；
* 排名类图表可按数值排序；

### 17.3 折线图

要求：

* 折线宽度统一；
* 数据点默认弱化或按需显示；
* 不使用发光线；
* 网格线使用 neutral border；
* 多系列使用受控序列色；
* hover 显示清晰 Tooltip。

### 17.4 面积图

如果当前支持：

* 面积透明度保持低；
* 不使用强渐变；
* 多系列避免大面积颜色覆盖；
* 深色模式保持可辨认。

### 17.5 饼图和环形图

要求：

* 分类数量过多时优先由现有路由切换柱状图；
* 扇区颜色使用受控 neutral 序列；
* 不使用 3D；
* 不使用外发光；
* 图例清晰；
* 标签避免重叠；
* 环形中心指标字体与 Astryx Typography 一致。

### 17.6 散点图

要求：

* 点大小适中；
* 不使用随机彩色；
* 多组数据使用受控系列色；
* 坐标轴和网格线弱化；
* Tooltip 显示维度和数值。

### 17.7 KPI 或指标卡

如果统一 Visualization Renderer 支持 KPI：

* 复用 Astryx Metric / Card；
* 不模拟图表；
* 不使用高饱和背景；
* 数值和单位层级清晰。

## 19. VisualizationSpec 与主题分离

`VisualizationSpec` 应描述：

* 图表类型；
* 数据来源；
* 维度；
* 指标；
* 标题；
* 图例；
* 格式化规则；
* 业务语义。

不应保存具体主题颜色。

错误：

```json
{
  "barColor": "#1677ff",
  "background": "#ffffff"
}
```

推荐：

```json
{
  "type": "bar",
  "theme": "neutral",
  "encoding": {
    "x": "五级分类",
    "y": ["贷款余额"]
  }
}
```

如果 `theme` 由客户端全局控制，也可以不在 Spec 中保存，统一由 Renderer 解析。

9. 不重复加载完整 SQL 或 Python 数据；

P0 至少完成 Artifact 缓存和稳定组件 key。

## 23. 图表节点稳定 Key

推荐：

```text
artifactId + reportVersion
```

作为节点稳定标识。

不得使用：

* 数组索引；
* Markdown 内容；
* 当前时间；
* 随机数；
* 图表加载状态。

避免报告滚动或主题变化时图表频繁卸载。


要求：

* 背景与报告页面一致；
* 坐标轴文字保持可读；
* 网格线不过亮；
* Tooltip 与页面主题一致；
* 系列颜色在深色背景有足够对比；
* 主题切换后图表即时更新；
* 不需要重新生成 Chart Artifact。

## 25. 可访问性

报告中的图表应包含：

* 可读标题；
* 简短描述；
* 图表类型；
* 数据摘要或文本替代；
* 不仅依靠颜色区分类别；
* Tooltip 不是唯一数据获取方式。

可以扩展：

```ts
export type VisualizationAccessibility = {
  ariaLabel: string;
  textSummary?: string;
};
```

如果当前 Renderer 已有 accessibility 能力，请直接复用。

## 26. 安全与数据边界

必须遵守：

1. 报告 Renderer 只能加载报告声明的 Chart Artifact；
2. 校验当前用户和会话权限；
3. 不允许 Markdown 节点传入任意文件路径；
4. 不允许节点执行脚本；
5. 不允许通过节点加载远端 URL；
6. 不暴露 SQLite 路径；
7. 不暴露临时文件路径；
8. 不展示完整原始数据；
9. Tooltip 继续遵守敏感字段脱敏策略；
10. Artifact 不存在时安全降级。

```ts
export type ReportVisualizationErrorCode =
  | 'VISUALIZATION_NODE_INVALID'
  | 'VISUALIZATION_ARTIFACT_NOT_FOUND'

  | 'VISUALIZATION_RENDER_FAILED'
  | 'CHART_THEME_RESOLVE_FAILED'
```

要求：

* 单个图表异常不影响报告正文；
* 错误提示面向用户；
* 技术堆栈仅记录日志；
* 不显示内部 Artifact ID；
* 可恢复时允许重试加载；
* 不自动重新执行 SQL 或 Python。
