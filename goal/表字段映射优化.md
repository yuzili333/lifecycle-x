## 1. 当前问题

当前 CSV 导入流程大致为：

```text
CSV 文件
→ 解析表头
→ 生成英文 SQLite 字段名
→ 导入 SQLite
→ 生成表结构清单
→ 注入 Schema Context
```

目前存在以下问题：

1. SQLite 物理表字段使用英文名称；
2. 报告 Skill 中定义的是中文业务字段名称；
3. 模型只能看到英文物理字段清单；
4. 英文字段名称无法稳定匹配 Skill 中的中文字段要求；
5. 不同 CSV 文件可能使用不同英文缩写；
6. 同一业务字段可能存在多个中文或英文别名；
7. Skill 与实际数据库物理字段名称直接耦合；
8. 字段推断结果未形成稳定、持久化的业务语义映射；
9. 数据管理页面中 CSV 表格列头存在固定文本“CSV推断字段”，无法准确反映字段业务含义。

需要将当前结构调整为：

```text
CSV 原始字段
→ SQLite 物理字段
→ 标准业务字段 ID
→ 中文展示名称
→ Skill 业务字段要求
```

## 6. 稳定的业务字段 ID

Skill、Schema Context 和字段解析逻辑必须依赖 `businessFieldId`，不能直接依赖中文名称或英文物理名称。

示例：

```text
credit.contract_id
credit.customer_id
credit.five_level_classification
credit.twelve_level_classification
credit.loan_balance
credit.contract_amount
credit.report_date
credit.institution_code
credit.institution_name
credit.product_code
credit.product_name
```

字段的职责划分：

| 属性                | 示例                    | 用途             |
| ----------------- | --------------------- | -------------- |
| `sourceHeader`    | 贷款余额（万元）              | 保留原 CSV 表头     |
| `physicalName`    | `loan_balance`        | SQLite 建表和 SQL |
| `businessFieldId` | `credit.loan_balance` | Skill 和语义解析    |
| `displayNameZh`   | 贷款余额                  | UI 和报告展示       |
| `fieldComment`    | 当前未偿还贷款本金余额           | Schema Context |

必须遵循：

```text
Skill 使用 businessFieldId
SQL 使用 physicalName
页面和报告使用 displayNameZh
```

## 12. 数据校验与导入策略

P0 支持：

```ts
export type CsvImportValidationMode =
  | 'strict'
  | 'quarantine';
```

### 12.1 strict

任何阻断型错误都会阻止整批正式导入：

* 必需字段缺失；
* 主键为空；
* 唯一字段重复；
* 类型无法转换；
* 非法枚举值；
* 超出数值范围；
* 字典和 CSV 表头不兼容。

### 12.2 quarantine

* 合法行写入正式临时表；
* 非法行写入错误记录或隔离数据集；
* 返回错误行号和错误字段；
* 用户需要明确确认后才能完成导入。

P0 不要求实现复杂宽松修复模式。

### 12.3 错误类型


## 19. Skill `overall-risk-classification-distribution` 改造

请优化本地预置 Skill：

```text
overall-risk-classification-distribution
```

### 19.1 禁止继续使用的方式

不要使用：

```yaml
requiredFields:
  - 合同编号
  - 五级分类
  - 十二级分类
  - 贷款余额
```

不要通过中文名称直接匹配 SQLite 字段。

### 19.2 新字段要求

改成：

```yaml
requiredFields:
  - businessFieldId: credit.contract_id
    displayNameZh: 合同编号
    required: true
    expectedTypes:
      - identifier
      - string

  - businessFieldId: credit.five_level_classification
    displayNameZh: 五级分类
    required: true
    expectedTypes:
      - category
      - string

  - businessFieldId: credit.twelve_level_classification
    displayNameZh: 十二级分类
    required: false
    expectedTypes:
      - category
      - string

  - businessFieldId: credit.loan_balance
    displayNameZh: 贷款余额
    required: true
    expectedTypes:
      - decimal
      - integer
      - number

  - businessFieldId: credit.contract_amount
    displayNameZh: 合同金额
    required: false
    expectedTypes:
      - decimal
      - integer
      - number
```

### 19.3 Skill 执行流程

```text
加载 Skill
→ 读取 requiredFields
→ 调用 BusinessFieldResolver
→ 校验必需字段
→ 获取实际 physicalName
→ 构造 SQL 工具参数
→ SQL 使用实际物理字段查询
→ 报告使用中文展示名称
```

### 19.4 SQL 工具输入

Skill 不应只传递中文字段名，应传递解析后的字段映射：

```ts
export type RiskClassificationResolvedFieldMapping = {
  contractId: {
    businessFieldId: 'credit.contract_id';
    physicalName: string;
    displayNameZh: string;
  };

  fiveLevelClassification: {
    businessFieldId: 'credit.five_level_classification';
    physicalName: string;
    displayNameZh: string;
  };

  twelveLevelClassification?: {
    businessFieldId: 'credit.twelve_level_classification';
    physicalName: string;
    displayNameZh: string;
  };

  loanBalance: {
    businessFieldId: 'credit.loan_balance';
    physicalName: string;
    displayNameZh: string;
  };

  contractAmount?: {
    businessFieldId: 'credit.contract_amount';
    physicalName: string;
    displayNameZh: string;
  };
};
```

### 19.5 降级规则

* 缺少 `credit.contract_id`：阻止完整报告生成；
* 缺少 `credit.five_level_classification`：阻止 Skill 执行；
* 缺少 `credit.loan_balance`：仅允许笔数分析，并明确限制；
* 缺少 `credit.twelve_level_classification`：跳过十二级分类分析；
* 缺少 `credit.contract_amount`：不输出合同金额合计；
* 映射存在歧义：要求用户确认，不得由模型自行选择。
