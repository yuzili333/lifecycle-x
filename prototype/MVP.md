# 存续期数据探针智能体 MVP 定义

## 1. MVP 目标

存续期数据探针智能体 MVP 面向银行贷后管理人员、后续尽职调查人员、业务分析人员和数据支持人员，优先验证“业务人员通过自然语言完成数据提取、数据清洗、数据加工、数据分析和分析材料生成”的核心价值。

MVP 不追求一次性覆盖所有贷后管理场景，也不由智能体直接输出风险信号、风险等级或处置建议。首期先聚焦高频、高价值、数据结构相对明确的数据处理与探索分析场景，验证以下假设：

1. 业务人员无需编写 SQL 或 Python，即可发起数据分析任务。
2. 系统能够从导入数据或连接数据源中识别关键字段和分析口径。
3. 系统输出的数据结果必须带有来源、口径和加工过程，能够被业务人员引用和复核。
4. 系统能够支持多轮追问和局部调整，而不是只输出一次性固定报表。
5. 系统能够生成可归档的 Markdown / PDF / JSON 结构化结果。

## 2. MVP 用户与场景

### 2.1 目标用户

| 用户类型 | 主要诉求 | MVP 覆盖程度 |
|---|---|---|
| 贷后管理人员 | 快速整理客户经营、回款、资金流向和担保相关数据 | 覆盖数据查询、加工和报告初稿 |
| 后续尽职调查人员 | 归集尽调材料、查看数据来源和分析口径 | 覆盖数据血缘和分析材料生成 |
| 风险管理人员 | 查看业务人员提交的数据分析依据 | 覆盖结构化数据结果和过程 trace |
| 数据分析/科技人员 | 降低临时取数和专项分析支持压力 | 覆盖 CSV/数据库接入的基础链路 |
| 审核/管理人员 | 复核报告依据与过程 trace | 覆盖归档和执行 trace 的基础能力 |

### 2.2 首期业务场景

MVP 首期只覆盖以下四类场景：

1. 资金流向数据整理：提取放款后账户流水、交易对手、交易摘要、金额和时间序列，形成可筛选明细和汇总结果。
2. 回款账户变化分析：按周期统计回款金额、回款频率、主要交易对手、资金留存时间等事实性指标。
3. 外部事项数据整理：对司法、工商、行政处罚、舆情等外部事项进行结构化归集和时间线整理，不自动判断风险等级。
4. 后续尽调材料初稿生成：根据数据来源、加工过程、统计口径和分析结果生成可复核的材料初稿。

## 3. MVP 范围

### 3.1 必须包含

| 模块 | MVP 能力 |
|---|---|
| 自然语言任务入口 | 支持输入分析目标、客户名称、时间范围、输出格式等自然语言需求 |
| 数据源导入 | 支持 CSV 文件导入；数据库连接作为可选验证能力 |
| 数据结构识别 | 识别字段名称、字段类型、金额字段、日期字段、客户字段、交易对手字段 |
| 分析计划生成 | 将用户意图拆解为数据选择、清洗规则、加工步骤、统计口径和输出物 |
| 数据清洗加工 | 支持字段映射、类型转换、缺失值提示、重复记录处理、分类汇总和周期对比 |
| 数据血缘生成 | 每项结果必须关联数据来源、记录范围、加工步骤和统计口径 |
| 多轮追问 | 支持基于上一轮结果调整时间范围、筛选客户、拆分维度、局部纠错 |
| 报告输出 | 支持 Markdown、JSON、PDF 报告初稿 |
| 归档引用 | 支持保存分析结果、报告、数据血缘和执行 trace |
| 异常处理 | 输入缺失、数据源失败、字段缺失、结果冲突、模型输出非法时返回结构化错误 |

### 3.2 暂不包含

| 暂不包含能力 | 说明 |
|---|---|
| 风险信号识别与风险等级判断 | 首期不由智能体输出风险信号、风险等级、预警结论或处置建议 |
| 自动发起审批或处置流程 | 不直接触发授信调整、提前收贷、风险处置等正式流程 |
| 全量外部数据实时采集 | 工商、司法、舆情等外部接口首期可通过导入数据验证 |
| 复杂机器学习模型训练 | 首期以数据加工、统计口径和可解释分析为主 |
| 多机构权限体系 | 首期只验证单机构或单租户使用流程 |
| 移动端 | 首期只覆盖桌面端操作台 |

## 4. MVP 核心用户流程

1. 业务人员选择或导入数据源。
2. 系统展示数据源字段、样例数据、识别出的客户字段、金额字段、日期字段。
3. 业务人员输入任务，例如：“提取 XX 公司近三个月放款后资金流向，按交易对手和交易摘要汇总，并生成尽调材料摘要。”
4. 系统生成分析计划，列出将使用的数据源、筛选条件、清洗规则、加工步骤和统计口径。
5. 业务人员确认或调整计划。
6. 系统执行分析并输出明细数据、汇总指标、数据血缘、图表和报告初稿。
7. 业务人员继续追问，例如“把时间范围扩展到半年”“按交易对手拆分”“剔除工资类交易”。
8. 系统基于上下文重新分析并保留版本记录。
9. 业务人员导出报告并归档。

## 5. MVP 成功标准

| 指标 | 验收标准 |
|---|---|
| 任务完成效率 | 典型资金流向整理任务从导入数据到报告初稿不超过 10 分钟 |
| 数据可追溯 | 100% 分析结果包含数据来源、记录范围和加工步骤 |
| 口径明确性 | 100% 指标包含统计口径、时间范围和筛选条件 |
| 异常可解释 | 字段缺失、数据源失败、结果冲突均返回结构化原因和补救建议 |
| 报告可用性 | 业务人员可基于报告初稿进行人工修订和归档 |
| 多轮可用性 | 至少支持 3 轮连续追问，且能引用上一轮结果 |

## 6. 首期样例数据要求

| 数据对象 | 关键字段 |
|---|---|
| 客户信息 | 客户编号、客户名称、统一社会信用代码、行业、实际控制人、关联企业 |
| 授信合同 | 贷款编号、贷款金额、贷款用途、放款日期、到期日期、还款方式、合同约束 |
| 账户流水 | 客户编号、账户号、交易日期、交易金额、借贷方向、交易对手、交易摘要、交易后余额 |
| 交易对手关系 | 交易对手名称、关系类型、是否关联方、是否个人账户、来源依据 |
| 外部事项 | 客户编号、事项类型、发生日期、事项描述、来源、影响范围 |
| 担保信息 | 押品编号、押品类型、估值、登记状态、查封冻结状态、担保覆盖率 |

## 7. MVP 风险与约束

1. 数据质量不足会影响分析结果，系统必须提示字段缺失和口径限制。
2. 大模型可能生成不可靠表述，报告生成必须只引用结构化分析结果和数据血缘。
3. 银行业务场景和口径存在机构差异，筛选条件、字段映射和统计口径需配置化。
4. 涉及客户、账户、资金流水等敏感数据，必须按银行内部数据安全规范处理。
5. MVP 输出只能作为数据分析材料，风险研判、业务定性和处置动作由业务人员完成。

## 8. MVP 交付物

1. 桌面端业务工作台原型。
2. 数据源导入与 schema 识别能力。
3. 自然语言任务到分析计划的转换能力。
4. 首批数据清洗、加工和指标口径。
5. 结构化分析结果、数据血缘、trace 和报告初稿。
6. 业务验收样例数据集。
7. MVP 试点反馈表和问题清单。
