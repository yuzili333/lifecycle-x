# 存续期数据探针智能体产品需求文档（PRD）

## 1. 产品概述

存续期数据探针智能体是一款面向银行贷后管理与后续尽职调查场景的桌面端业务分析工具。产品通过自然语言交互、多源数据接入、数据清洗、数据加工、探索分析、数据血缘生成和报告输出，帮助业务人员从依赖固定报表和专项取数转向自主完成数据分析材料准备。

产品首期以 MVP 验证为目标，优先支持 CSV 导入和可配置数据源，围绕资金流向提取、回款变化统计、外部事项整理、担保数据汇总和尽调材料初稿生成形成闭环。经业务方确认，首期不由智能体输出风险信号、风险等级、预警结论或处置建议。

### 1.1 业务背景与价值

贷款存续期分析涉及客户、贷款、账户流水、交易对手、担保和外部事项等多类数据。传统处理方式依赖跨系统取数、Excel 加工、SQL 查询和人工报告编写，容易出现处理耗时、字段口径不一致、分析过程难复用、结果依据难追溯等问题。

产品通过自然语言任务入口和受控工具链，将“任务理解—数据选择—字段与口径确认—查询—数据集物化—统计分析—可视化—报告—Artifact 归档”串成可审计流程，核心价值是降低技术门槛、提升分析效率、统一口径、支持多轮探索、规范报告材料并释放科技人员的重复支持压力。

### 1.2 产品定位与边界

- 产品是事实性数据分析辅助工具，不是自动风险决策、审批或处置系统。
- 可以查询和分析数据源中已有的风险分类、预警标识等事实字段，但不得自行生成新的风险信号、风险等级或处置建议。
- 模型只负责理解、规划、生成受控工具参数和整理已有结果；SQL/Python/Artifact 才是权威结果来源。
- 模型上下文只接收必要的 Schema、字段映射、摘要和 Artifact 引用，不接收完整源数据或数据库凭据。
- 数据源失败时不得静默替换为模拟数据或未选择的数据源。

### 1.3 当前版本摘要

截至 2026-07-23，当前代码已经形成 Electron 工作台、认证与权限、会话、模型配置、持久/临时 CSV、Schema 上下文、Tool/Agent 编排、受控 SQL、受控 Python、图表、Markdown 报告、Artifact 血缘、版本、审计、本地 Memory 和一个预置业务 Skill 的基础闭环。真实 MySQL 连接、PDF/JSON 文件导出、独立血缘/报告/审计页面以及多数业务场景专用模板尚未完成。

## 2. 产品目标

1. 降低业务人员开展数据分析的技术门槛。
2. 提升贷后检查和后续尽调的分析效率。
3. 将分析结果与数据来源、加工步骤和统计口径绑定，提升可复核性。
4. 支持多轮探索式分析，适配业务人员动态调整思路的工作方式。
5. 形成可归档、可引用、可审计的分析资产。

## 3. 用户画像

|---|---|---|---|

| 科技/数据人员 | 低频 | 提供数据接入和字段口径支持 | 临时取数需求多 |

## 4. 产品范围

### 4.1 首期范围

- 桌面端工作台。
- 密码/SSO 登录流程、会话管理、角色权限和个人设置。
- 自然语言任务输入。
- 持久 CSV 和会话临时 CSV 数据导入；持久 CSV 同时校验表字典。
- 数据源字段识别和字段映射。
- 分析计划展示。
- 受控 SQL/Python 工具调用、审批、取消和审计。
- 资金流向、回款变化、外部事项、担保数据等事实性数据提取和分析。
- 关键指标、明细结果、数据血缘、待复核事项输出。
- 多类型图表与 Markdown 报告 Artifact 生成。
- 历史结果归档和引用。
- 结构化异常返回和执行 trace。
- 本地 Memory、上下文压缩、敏感内容过滤和预置 Skill。

### 4.2 非首期范围

- 直接写入银行核心业务系统。
- 自动审批、自动预警下发、自动风险分类调整。
- 智能体自动输出风险信号、风险等级、风险定性或处置建议。
- 全量实时外部数据采集。
- 跨机构多租户复杂权限。
- 移动端应用。
- 复杂模型训练平台。
- 当前版本中的 PDF/JSON 文件导出、独立报告库、独立数据血缘库和独立审计追踪页面。
- 当前模拟 MySQL 适配器之外的真实生产数据库接入，需完成专项安全与联调验收后再纳入交付范围。

## 6. 核心功能需求

### 6.1 数据源管理

#### 功能说明

业务人员或数据管理员可导入 CSV 文件，或选择已配置的数据库连接。系统读取字段、样例数据和数据质量摘要，并提示关键字段映射情况。

#### 主要交互

1. 上传 CSV 文件。
2. 查看字段识别结果。
3. 修正字段映射。
4. 保存数据源。
5. 查看数据质量提示。

#### 验收标准

- 能识别文本、日期、金额、枚举、编号等常见字段类型。
- 能提示客户字段、贷款字段、金额字段、日期字段、交易对手字段是否缺失。
- 字段缺失时给出明确补充建议。
- 数据源应生成唯一来源编号。

### 6.2 自然语言任务入口

#### 功能说明

用户通过自然语言描述分析目标，系统识别任务类型、客户对象、时间范围、输出格式和需要引用的历史结果。

#### 示例输入

- “提取 XX 公司放款后三个月资金流向，按交易对手和交易摘要汇总，并输出数据血缘。”
- “分析该客户近半年回款是否下降，按主要交易对手拆分。”
- “基于本次数据分析结果生成后续尽调材料初稿。”

#### 验收标准

- 能识别资金流向整理、回款分析、外部事项整理、担保数据汇总、报告生成等任务类型。
- 关键条件缺失时不直接执行，应要求用户补充。
- 能基于上一轮分析结果继续追问。

### 6.3 分析计划生成

#### 功能说明

系统在执行前展示分析计划，包含数据源、字段映射、清洗规则、筛选条件、加工步骤、统计口径和输出物。

#### 计划内容

| 内容 | 说明 |
|---|---|
| 任务类型 | 资金流向整理、回款分析、外部事项整理等 |
| 数据来源 | 参与分析的数据源编号 |
| 字段映射 | 客户、贷款、账户、日期、金额、交易对手等字段的映射关系 |
| 清洗规则 | 类型转换、缺失值处理、去重、标准化等处理规则 |
| 筛选条件 | 客户、贷款、时间范围等 |
| 统计口径 | 计算公式、对比周期、排除规则 |
| 加工步骤 | 明细提取、关联、分组汇总、周期对比、排序等步骤 |
| 输出物 | JSON、Markdown、PDF、图表 |

#### 验收标准

- 执行前可查看计划。
- 计划中不得出现无法解释的数据来源或口径。
- 用户可调整时间范围、客户范围和输出格式。

### 6.4 数据清洗与加工

#### 功能说明

系统根据用户任务和字段映射，对导入数据进行清洗、关联、转换、汇总和重算。首期只输出事实性加工结果，不输出风险信号、风险等级或处置建议。

#### 首批加工能力

| 能力 | 处理示例 | 输出要求 |
|---|---|---|
| 字段标准化 | 日期格式、金额格式、客户名称、交易对手名称标准化 | 字段映射、转换结果、异常字段清单 |
| 明细筛选 | 按客户、贷款、账户、时间范围、交易摘要筛选流水 | 明细数据、筛选条件、来源编号 |
| 数据关联 | 关联贷款、客户、账户、交易对手、担保和外部事项数据 | 关联字段、匹配结果、未匹配记录 |
| 分组汇总 | 按交易对手、账户、月份、摘要分类统计金额和笔数 | 汇总表、计算公式、统计口径 |
| 周期对比 | 对比本期与上期回款金额、频率、交易对手变化 | 对比结果、周期定义、排除规则 |

#### 验收标准

- 每项加工结果必须包含数据来源、处理步骤和统计口径。
- 字段缺失、格式异常、未匹配记录应单独输出待复核清单。
- 系统不得将加工结果直接表述为风险结论或处置建议。

### 6.5 探索分析与指标输出

#### 功能说明

系统根据用户任务生成事实性分析结果，例如资金流向汇总、回款周期对比、交易对手分布、外部事项时间线和担保数据变化表。

#### 分析结果类型

| 类型 | 说明 |
|---|---|
| 明细结果 | 满足筛选条件的原始或清洗后记录 |
| 汇总指标 | 金额、笔数、频率、占比、环比等统计结果 |
| 对比结果 | 不同周期、账户、交易对手或数据源之间的差异 |
| 时间线 | 外部事项、担保变化、资金流转等按时间排序的事实列表 |
| 待复核事项 | 数据缺失、口径冲突、字段无法匹配或来源不一致的事项 |

#### 验收标准

- 分析结果只描述数据事实和统计结果。
- 不输出“高风险”“预警”“建议处置”等业务定性。
- 用户可继续调整筛选条件、拆分维度或排除规则并重新计算。

### 6.6 数据血缘管理

#### 功能说明

数据血缘用于支撑分析结果和报告内容，确保每个指标、明细和报告片段可复核、可引用。

#### 数据血缘字段

| 字段 | 说明 |
|---|---|
| 结果编号 | 唯一标识分析结果或报告片段 |
| 数据来源编号 | 指向 CSV、数据库表或历史 artifact |
| 查询编号 | 指向本次分析中的查询或统计步骤 |
| 原始记录范围 | 指向原始记录或记录集合 |
| 加工步骤 | 指向清洗、关联、汇总、排序、排除等处理步骤 |
| 时间范围 | 结果覆盖的日期范围 |
| 统计口径 | 指标计算方式和筛选条件 |
| 结果摘要 | 面向业务人员的事实描述 |

#### 验收标准

- 指标和报告内容都能引用结果编号。
- 结果摘要不能脱离原始数据产生新事实。
- 多源冲突应形成冲突 trace。

### 6.7 报告生成

#### 功能说明

系统基于结构化结果生成报告初稿，支持业务人员下载、修订和归档。

#### 报告类型

1. 存续期数据分析摘要。
2. 资金流向分析材料。
3. 后续尽调数据材料初稿。
4. 到期续贷数据分析材料初稿。

#### 报告章节

- 客户与贷款概况。
- 分析范围和数据来源。
- 关键指标。
- 明细结果和汇总结果。
- 数据血缘摘要。
- 待复核数据问题。
- 数据缺口和人工复核事项。

#### 验收标准

- 报告中的事实必须能追溯到结果编号和数据来源。
- 报告必须展示数据范围和统计口径。
- 数据不足时必须写明影响范围。
- 报告不得自动生成风险等级、风险定性或处置建议。

### 6.8 历史上下文与多轮分析

#### 功能说明

用户可基于历史分析继续追问、纠错和生成报告。

#### 示例

- “把上次结果中关联方交易按交易对手排序。”
- “剔除工资、税费和社保交易后重新计算。”
- “把这次分析结果追加到报告。”

#### 验收标准

- 系统能识别用户引用的历史任务或 artifact。
- 每次重算都生成新的版本记录。
- 局部纠错不得覆盖原始结果，应保留版本差异。

### 6.9 异常处理与审计追踪

#### 功能说明

系统应记录任务执行过程中的失败、重试、降级、冲突和模型输出校验结果。

#### 异常类型

| 异常 | 处理规则 |
|---|---|
| 输入缺失 | 返回缺失字段和补充建议 |
| 数据源失败 | 重试；失败后记录原因；可降级时标记部分成功 |
| 字段缺失 | 提示手动映射或补充数据 |
| 多源冲突 | 输出冲突项和复核建议 |
| 输出格式非法 | 重新校验或返回结构化错误 |
| 分析无法完成 | 返回失败阶段、已完成步骤和下一步建议 |

#### 验收标准

- 所有异常应有错误码、可恢复标识和 trace 编号。
- 降级结果必须标记为部分成功。
- 数据不足、口径冲突或来源不一致时输出待复核事项，不输出风险定性。

| Markdown 报告 | 报告 Artifact、版本、流式报告卡片、图表引用、查看和复制 | PDF/JSON 文件导出、正式模板、导出审计尚未实现 |

| 模型运行配置 | OpenAI-compatible 流式适配、模型/执行模型、双模型编排、Thinking 优化、API Key 本地配置 | 外部模型使用仍需完成数据出域、脱敏和合规确认 |

## 7. 页面需求

### 7.1 工作台页面

核心区域：

- 自然语言任务输入框。
- 数据源选择器。
- 分析计划卡片。
- 关键指标区。
- 明细与汇总结果区。
- 数据血缘侧栏。
- 报告 Artifact 查看与复制入口；PDF/JSON 文件导出按钮为后续能力。
- 执行 trace 抽屉。

关键状态：

- 空状态：提示导入数据源或选择历史数据。
- 计划待确认：展示分析计划和可调整项。
- 执行中：展示当前步骤。
- 成功：展示明细、指标、数据血缘和报告入口。
- 部分成功：展示已完成结果和数据缺口。
- 失败：展示结构化错误和补救建议。

## 8. 数据与协议需求

### 8.1 任务请求

任务请求应包含任务编号、用户意图、数据源编号、上下文 artifact、客户或贷款约束、时间范围和输出格式。

### 8.2 分析结果

分析结果应包含任务状态、摘要、明细结果、汇总指标、数据血缘、待复核事项、artifact 和 trace。

### 8.3 错误返回

错误返回应包含错误码、错误说明、是否可恢复、是否可重试、是否使用降级方案和 trace 编号。

## 9. 权限与安全需求

1. 数据库账号、客户信息、账户流水、征信和司法数据属于敏感信息，应按银行内部标准加密、脱敏和审计。
2. 桌面端不得明文保存数据库密码。
3. 用户只能访问授权数据源。
4. 报告导出应记录导出人、导出时间、报告编号和数据范围。
5. 外部模型或服务使用前需确认数据出域和脱敏策略。

当前版本已实现认证/角色权限、数据源权限校验、SQL/Python 审批与审计、敏感字段脱敏以及本地模型 API Key 配置；报告文件导出及其审计、生产级数据库凭据管理和银行统一身份接入仍待完成。

## 10. 非功能需求

| 类型 | 要求 |
|---|---|
| 性能 | MVP 典型 CSV 分析任务应在 10 分钟内完成，具体数据量上限待补充 |
| 可用性 | 关键错误应给出业务可理解的补救建议 |
| 可追溯 | 所有指标、明细结果和报告均需关联数据来源、加工步骤或 trace |
| 可配置 | 报告模板、字段映射、统计口径和清洗规则应支持配置 |
| 可扩展 | 后续可扩展数据库、Excel、外部接口、指标库和模型能力 |
| 合规 | 不替代人工决策，不输出无数据支持的事实或风险定性，不绕过内部审批流程 |

性能、100% 血缘覆盖和真实用户可用性属于业务验收指标。当前自动化测试只能证明已编码的安全与流程断言，不能替代真实数据规模、真实数据库、正式权限和试点用户的端到端验收。

## 11. 验收用例

| 用例 | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| 资金流向提取 | 已导入客户、贷款、流水、交易对手数据 | 输入资金流向整理任务 | 输出交易明细、交易对手汇总、金额合计、数据来源和统计口径 |
| 回款周期统计 | 已导入连续周期流水 | 输入回款变化分析任务 | 输出各周期回款金额、笔数、对比结果、统计口径和数据血缘 |
| 外部事项整理 | 已导入外部事项数据 | 输入外部事项整理任务 | 输出事项类型、发生日期、来源、时间线和原始记录范围 |
| 字段缺失处理 | 流水数据缺少交易日期 | 发起回款分析 | 返回结构化错误，提示补充或映射交易日期字段 |
| 多轮追问 | 已完成一次分析 | 输入“按交易对手拆分” | 基于上一轮结果重新生成拆分结果和新版本 |
| Markdown 报告生成 | 已完成数据分析 | 选择生成报告 | 生成 Markdown 报告 Artifact，包含数据来源、分析结果、图表引用、血缘和复核事项 |
| PDF/JSON 导出 | 已生成报告 Artifact | 查找或尝试使用 PDF/JSON 导出 | 当前版本不提供可执行入口或明确标记为待实现，不得伪造下载成功 |
| SQL 审批与拒绝 | 已生成受控只读 SQL 请求 | 批准或拒绝工具调用 | 批准后执行并记录审计；拒绝后不执行且保留状态 |
| Python 数据集边界 | 已形成受控 Workflow Dataset | 发起 Python 分析 | Python 只消费已确认数据集，不连接业务数据库 |
| 临时 CSV 隔离 | 会话中已附加临时 CSV | 切换会话或删除附件 | 临时数据仅在所属会话可用，不自动转为持久数据源 |

## 12. 业务待确认问题

1. 首期试点是否以公司贷款、流动资金贷款为主。
2. 放款后首次检查的默认时间范围是 7 天、15 天、30 天还是其他。
3. 回款变化、资金流向、担保数据等指标的统计口径和默认周期。
4. 关联方名单由系统导入、人工维护还是从外部工商关系识别。
5. 报告模板是否需要符合现行贷后检查报告格式。
6. PDF 报告是否需要电子签章、二维码或防篡改水印。
7. 数据归档期限和历史分析结果复用范围。
8. 是否允许接入外部大模型，是否必须本地化部署。
9. 哪些内容只能由业务人员人工研判，禁止由智能体自动输出。
