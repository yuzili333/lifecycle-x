# 生产环境离线数据探索智能体需求方案

## 1. 项目概述

1.1 项目背景

当前公司内部数据分析工作存在典型的业务与技术断层痛点：

•业务人员：熟悉业务指标、分析诉求、业务逻辑，但无Python、SQL编程能力，无法自主对业务大宽表进行数据探查、统计分析、可视化出图；

•研发/数据分析师：掌握代码、数据分析工具，但不熟悉底层业务口径、业务分析侧重点，需要反复和业务对齐需求；

•行业共性痛点：存续期管理环节业务数据分析大多为递进式探索性分析，无固定分析脚本；分析思路、筛选条件、统计维度、图表样式会根据上一轮输出结果动态调整、临时纠错、切换分析方向；现有固定报表、离线脚本无法适配动态多轮探索场景。

基于以上现状，拟搭建部署在公司内网生产环境的离线数据探索智能体，无云端大模型调用能力，依托生产环境本地离线大模型，通过自然语言多轮对话，实现业务人员自主完成大宽表探索、多表关联、统计分析、图文报告输出。

1.2 项目定位

轻量化生产级离线数据分析智能对话机器人，定位为业务人员专属无代码数据分析助手；仅依托本地离线模型+基础办公、Python工具运行，不联网、不调用任何云端大模型接口；主打多轮递进式探索分析，支持基于历史对话和上一轮分析结果迭代优化分析任务。

1.3 约束前提（硬性红线，研发必须严格遵守）

1.网络与模型约束（最高优先级）：生产环境严禁调用任何云端、公有云大模型API；所有大模型推理、智能体逻辑全部依赖公司生产环境已部署的主流离线开源大模型完成，全程内网闭环、无外网出网流量；

2.工具约束：智能体仅允许调用三类基础工具，不接入第三方复杂数据分析插件、外部接口：① 生产环境Python运行环境（数据分析、数据清洗、统计计算、绘图）；② WPS办公套件；③ Microsoft Office办公套件；

3.数据边界约束：仅对接公司生产业务库业务大宽表、业务基础数据表；遵循公司生产数据权限体系，智能体继承操作用户的数据权限；

4.运行环境约束：全功能部署、运行、存储全部在公司内网生产集群，不得迁移测试环境、外网环境。

## 2. 核心解决痛点

1.解决业务－技术人员信息壁垒：业务人员纯自然语言交互，零代码完成数据分析，无需协调数据研发排期；研发人员无需承接零散、临时的探索性数据查询需求；

2.解决静态报表无法适配动态探索分析问题：区别于固定报表，支持分析思路动态变更：根据上一轮图表、数据结果，调整筛选条件、拆分维度、更换统计算法、切换图表类型、局部纠错；

3.解决单次任务无法迭代延续问题：支持长上下文多轮对话，全部历史分析数据、对话逻辑、中间结果全程留存，后续任务基于历史上下文叠加执行；

4.轻量化落地：不新增复杂中间件、不接入繁多外部技能，仅依托现有Python+Office/WPS基础工具落地，降低生产环境改造和运维成本。

## 3. 整体业务流程

1. 用户登录生产环境离线数据探索智能体页面，系统自动同步用户生产数据权限；

2. 用户自然语言输入首轮分析诉求（针对指定大宽表做数据探索）；

3. 离线大模型解析语义、自动编排Python工具执行分析任务；

4. 智能体输出结构化文字结论+可视化图表；

5. 用户基于输出结果，多轮对话下发迭代指令：微调条件、切换分析方法、多表关联、修正错误数据、更换图表；

6. 多轮迭代完成后，智能体导出Office/WPS格式图文分析报告；

7. 全程留存对话日志、执行链路、分析中间数据。

## 4. 详细功能需求（优先级P0必须上线，P1迭代上线）

### 4.1 P0 核心刚需功能（一期必须全部落地）

#### 4.1.1 长上下文多轮对话交互能力（核心能力）

•支持导入宽表功能，业务人员能够导入一张或多张数据表作为分析基础。

•支持会话级上下文记忆，智能体全程记忆当前对话内容的已选定数据表、筛选口径、分析维度、历史执行结果、用户偏好。

•支持基于上一轮分析结果递进执行新任务：局部下钻分析、缩小/扩大数据范围、变更统计指标、修正错误分析逻辑；

•支持用户口头纠错：识别自然语言纠错指令，回滚上一步操作、修正错误口径、剔除异常脏数据、重新计算；

•支持同会话内切换分析目标：原有数据表不变，直接切换分析侧重点；保留历史会话记录，防止丢失。

•支持新建多组独立会话，会话之间数据、上下文互相隔离。

#### 4.1.2 大宽表智能数据探索分析

•核心适配对象：公司生产环境海量字段业务大宽表；

•语义解析能力：识别业务口语化指标、业务字段含义、业务筛选口径（无需用户写字段名、SQL语句）；

•基础探索能力：数据抽样探查、字段分布分析、异常值筛查、空值统计、数据趋势探查、维度拆分；

•自动识别大宽表数据表结构、字段释义、数据量级，前置规避超量查询导致生产集群压力过大问题。

#### 4.1.3 基础数据处理与统计分析能力

•常规统计计算：求和、平均值、中位数、极值、占比、同比/环比、分组聚合、排序排名；

•简单数据清洗：异常值过滤、空值剔除、重复数据去重；

•简单多表关联能力：支持用户自然语言描述关联规则，完成生产环境多张业务表内连接、左连接、字段合并；支持自定义关联主键，无需手写关联语句；仅限简单业务多表关联，对复杂多层笛卡尔乘积查询做权限拦截。

4.1.4 自动可视化图表生成

•智能根据分析场景、数据类型自动匹配最优图表类型：折线图、柱状图、饼图、散点图、统计表、热力图。

•支持多轮对话修改图表：调整坐标轴、图例、图表标题、颜色、维度展示顺序、合并子图表；

•图表文件存储在本地生产服务器，可预览、单独下载。

4.1.5 图文结论生成与办公工具导出

•自动总结数据分析结果，输出业务化文字分析结论（规避代码话术，输出业务人员看得懂的总结）；

•将文字结论与原生图表整合，调用WPS/Office工具，导出Word、Excel标准分析报告；

•支持用户对话调整报告文案、补充分析备注、删减冗余结论。

4.1.6 工具调用编排能力

•智能体自主判断任务类型，自动调度生产环境Python环境完成计算、绘图、数据查询；

•任务收尾自动调度WPS/Office完成文件排版、导出；

•对用户透明：业务人员无感后台调用工具，只接收最终分析结果。

4.2 P1 迭代优化功能（二期上线）

•会话历史任务复刻：一键复刻历史对话分析流程，替换数据表并批量复用分析逻辑；

•常用分析口径收藏：保存高频业务筛选条件、统计模型；

•生产环境任务限流：对大宽表超大批量查询自动分片执行，保护生产数据库性能。

•简单数据下钻联动：图表联动底层明细数据查看。

## 5. 非功能需求（研发硬性落地标准）

5.1 部署与模型约束（最高优先级）

•全程纯离线闭环：禁止任何形式调用阿里云、腾讯云、OpenAI等公有云大模型接口；禁止生产环境节点外网访问。

•模型底座：仅使用公司生产环境已完成部署的主流离线开源大模型作为语义解析、对话编排、任务决策底座；不新增大模型训练、微调集群。

•无第三方外部技能插件：仅调用原生Python、WPS、Office，禁止接入爬虫、外部数据源、第三方BI插件。

5.2 生产环境性能要求

•上游大宽表单表支持千万级数据量正常探索分析。

•常规分析任务响应时长≤20s，复杂多表关联聚合任务响应时长≤60s。

•增加生产资源熔断机制：防止智能体大查询打垮生产业务数据库，超限自动终止任务并推送提示。

•多用户并发：支持部门内10人同时在线会话分析，不互相抢占资源。

5.3 数据安全与权限

•严格继承公司生产系统人员数据权限：用户只能查询、分析自己权限范围内的数据表和字段；

•将全部对话日志、分析中间数据、导出报告留存于生产内网服务器，留存周期90天，支持运维审计；

•禁止数据外发，禁止下载到外网终端；文件仅在可在内网终端预览和下载。

5.4 交互体验

•极简对话式UI，业务人员零学习成本；

•后台屏蔽全部代码、SQL、执行日志；前端只展示对话、图表、文字报告；

•执行失败时进行通俗化报错：不输出代码报错堆栈，输出业务通俗原因及修正建议。

## 8. 项目边界&amp；不做功能（明确排期边界，避免需求蔓延）

1.不调用任何公有云、云端大模型、外部AI插件；

2.不接入BI大屏、数据库以外第三方数据源；

3.不支持复杂多层嵌套多表关联、跨库复杂 join 任务；仅支持简单同库关联；

4.不做机器学习预测、算法建模；仅做探索、统计、可视化分析；

5.不新增额外运维中间件，不改造现有生产数据库结构；

6.不开放自定义代码输入端口，业务人员无法执行高危命令，保障生产安全。
