# 银行贷款后续尽职调查中的存续期业务背景说明

## 1. 文档目的

本文档用于补充“存续期数据探针智能体”项目的业务背景，帮助产品、研发、算法、风控及业务团队理解银行贷款后续尽职调查中的“存续期业务”含义、管理目标、典型场景、数据对象和智能体应用价值。

“存续期数据探针智能体”面向银行贷后管理与后续尽职调查场景，重点解决贷款发放后客户经营变化、资金用途异常、担保状态变化、还款能力下降、风险预警滞后、尽调证据分散等问题，通过多源数据整合、风险信号识别、证据链生成和智能分析建议，提升存续期业务管理效率和风险识别能力。

---

## 2. 存续期业务的定义

在银行贷款业务中，贷款通常经历以下生命周期：

```text
贷前调查 → 授信审批 → 合同签署 → 放款执行 → 存续期管理 → 到期收回 / 续贷 / 展期 / 重组 / 风险处置
```

其中，**存续期业务**是指：

> 一笔贷款或授信从放款、额度生效或业务落地开始，到贷款结清、额度终止、风险处置完成之前，银行围绕该笔业务开展的持续管理、风险监测、后续尽职调查和处置活动。

存续期业务并不只是“贷款已经发放后的存量业务”，而是银行对贷款风险进行持续识别、持续验证和持续管理的完整过程。

在该阶段，银行需要持续判断以下问题：

1. 借款人是否仍具备稳定的偿债能力；
2. 贷款资金是否按照合同约定用途使用；
3. 第一还款来源是否稳定；
4. 第二还款来源是否充足、有效；
5. 抵质押物或保证担保是否发生弱化；
6. 借款人经营、财务、信用、司法、舆情等是否发生重大变化；
7. 贷款风险分类是否需要调整；
8. 是否需要采取压降授信、追加担保、提前收贷、展期、重组或风险处置等措施。

---

## 3. 存续期业务与贷后管理、后续尽职调查的关系

在银行信贷管理语境中，“存续期业务”“贷后管理”“后续尽职调查”三个概念密切相关，但侧重点不同。

| 概念     | 业务定位   | 主要含义                                |
| ------ | ------ | ----------------------------------- |
| 存续期业务  | 生命周期阶段 | 指贷款发放后至结清、终止或处置前的全部在贷期间业务           |
| 贷后管理   | 管理体系   | 指银行对贷款资金、客户经营、还款来源、担保状态、风险分类等进行持续管理 |
| 后续尽职调查 | 调查方法   | 指银行在贷款存续期间，对客户、资金、担保、风险变化等进行再次核查和验证 |

可以概括为：

> 存续期业务是业务阶段，贷后管理是管理体系，后续尽职调查是其中的重要工作方法。

例如，某企业获得流动资金贷款后，银行在贷款存续期发现其账户流水下降、征信新增逾期、主要客户回款减少、抵押物涉及查封。此时，银行需要开展后续尽职调查，核实企业真实经营状况、资金用途、还款能力和担保有效性，并根据调查结果调整风险策略。

## 4. 存续期业务的核心管理目标

银行开展存续期业务管理，核心目标包括以下几个方面。

### 4.1 确认贷款资金用途合规

银行需要核实贷款资金是否按照合同约定用途使用。例如：

* 流动资金贷款是否用于日常经营周转；
* 固定资产贷款是否用于项目建设；
* 贸易融资资金是否与真实交易背景匹配；
* 贷款资金是否流入关联方；
* 是否存在资金空转、回流、挪用；
* 是否违规流入房地产、证券、股权投资等限制领域。

### 4.2 持续评估客户还款能力

贷款发放后，借款人的经营状况可能发生变化。银行需要持续评估其偿债能力，包括：

* 营业收入变化；
* 经营现金流变化；
* 纳税情况变化；
* 应收账款回款情况；
* 资产负债率变化；
* 新增融资情况；
* 主要客户或供应商变化；
* 行业景气度变化；
* 实际控制人和管理层变化。

### 4.3 监控担保和抵质押物状态

贷款审批时担保充足，并不代表存续期间一直安全。银行需要关注：

* 抵押物是否贬值；
* 抵押物是否被查封、冻结或处置；
* 抵押物权属是否发生变化；
* 是否存在重复抵押；
* 保证人代偿能力是否下降；
* 质押物价值是否波动；
* 担保合同和登记手续是否持续有效。

### 4.4 识别合同违约和风险触发事项

银行需要监测借款人是否违反授信合同、借款合同或相关承诺。例如：

* 未按约定用途使用贷款；
* 未按时还本付息；
* 未按约定提供财务资料；
* 擅自进行重大投资、并购或资产处置；
* 实际控制人发生重大变化；
* 重大诉讼、仲裁或行政处罚；
* 经营恶化或停工停产；
* 其他银行贷款出现逾期或交叉违约。

### 4.5 支持风险分类和资产质量管理

存续期管理结果会影响贷款风险分类。银行需要根据客户风险变化，判断贷款是否仍属于正常类，或是否需要调整为关注类、次级类、可疑类或损失类。

风险分类不仅影响单笔贷款管理，也影响银行整体资产质量、拨备计提、资本占用和风险管理策略。

## 5. 存续期业务的典型场景

### 5.1 放款后首次检查

#### 场景说明

贷款发放后，银行需要在较短时间内核实贷款资金是否按照审批用途和合同约定使用。

#### 重点核查内容

* 放款账户流水；
* 受托支付记录；
* 借款人自主支付记录；
* 收款方信息；
* 采购合同、发票、物流单据；
* 交易对手是否真实；
* 资金是否回流借款人或关联方账户；
* 是否存在异常拆分支付、循环转账或资金空转。

#### 智能体应用价值

存续期数据探针智能体可自动比对：

```text
审批用途 → 合同约定 → 实际资金流向 → 交易凭证 → 收款方关系
```

帮助识别资金用途不一致、资金回流、异常交易对手、虚假贸易背景等风险。

---

### 5.2 定期贷后检查

#### 场景说明

在贷款存续期间，银行通常需要按照月度、季度、半年度或年度开展贷后检查。

#### 重点核查内容

* 企业经营是否正常；
* 主营业务收入是否变化；
* 账户流水是否萎缩；
* 纳税规模是否下降；
* 应收账款是否异常增加；
* 存货是否异常积压；
* 资产负债率是否上升；
* 征信是否新增逾期；
* 是否存在新增大额融资；
* 是否涉及司法诉讼、被执行或行政处罚。

#### 智能体应用价值

智能体可整合财务、流水、征信、工商、司法、税务、舆情等多源数据，自动生成：

```text
本期经营变化摘要
本期风险信号
异常数据证据
风险等级建议
后续尽调建议
```

---

### 5.3 资金回笼账户监测

#### 场景说明

银行通常会要求借款人使用指定账户作为销售回款账户或资金回笼账户，以便观察企业真实经营现金流。

#### 重点核查内容

* 销售回款是否进入约定账户；
* 回款金额是否与历史水平匹配；
* 回款频率是否明显下降；
* 到账资金是否快速转出；
* 是否频繁转入个人账户；
* 是否存在大额异常支出；
* 是否存在与主营业务无关的交易；
* 是否存在资金被其他债权人划扣的情况。

#### 智能体应用价值

智能体可对账户流水进行结构化分析，识别：

* 回款下降；
* 资金快进快出；
* 大额异常转账；
* 交易对手异常；
* 关联方资金往来；
* 疑似资金空转；
* 经营现金流与报表收入不匹配。

---

### 5.4 固定资产贷款或项目贷款进度跟踪

#### 场景说明

对于固定资产贷款、项目贷款或项目融资，银行需要持续核实项目建设进度、资本金到位情况和贷款资金使用进度。

#### 重点核查内容

* 项目资本金是否按比例到位；
* 工程建设是否按计划推进；
* 设备采购是否真实；
* 施工合同、发票、付款凭证是否一致；
* 贷款支用是否与项目进度匹配；
* 是否存在项目停工、延期、超预算；
* 是否存在贷款资金超前支付或挪用。

#### 智能体应用价值

智能体可建立项目进度与资金支用的对比关系：

```text
项目计划进度 → 实际工程进度 → 资本金投入 → 贷款支用 → 合同/发票/验收资料
```

帮助识别项目停滞、资本金不到位、贷款超进度支用、工程资料异常等风险。

---

### 5.5 企业重大事项触发后续尽调

#### 场景说明

借款人在贷款存续期间发生重大事项时，银行需要重新评估其偿债能力和授信风险。

#### 常见触发事项

* 股权结构变更；
* 实际控制人变更；
* 高管团队重大调整；
* 重大诉讼或仲裁；
* 被列为被执行人或失信被执行人；
* 核心资产被查封、冻结；
* 对外担保大幅增加；
* 行政处罚或环保处罚；
* 重大负面舆情；
* 主要客户流失；
* 上下游供应链中断；
* 其他银行贷款出现逾期。

#### 智能体应用价值

智能体可持续监听外部数据源变化，自动触发：

```text
重大事项预警
后续尽调任务
风险重新评估
授信策略调整建议
```

---

### 5.6 贷款到期前续贷、展期或退出判断

#### 场景说明

贷款到期前，银行需要判断客户是否具备按期还款能力，是否可以续贷、展期、借新还旧、无还本续贷，或是否应压降退出。

#### 重点核查内容

* 当前经营是否正常；
* 到期还款资金来源是否明确；
* 现金流是否覆盖本息；
* 是否依赖新增融资维持周转；
* 是否存在隐性逾期；
* 是否存在高频倒贷；
* 担保是否仍然足值；
* 行业风险是否上升；
* 银行内部风险策略是否变化。

#### 智能体应用价值

智能体可辅助生成到期风险评估报告，给出策略建议：

```text
正常续贷
压降续贷
追加担保
展期观察
提前退出
风险处置
```

---

### 5.7 风险预警与问题贷款管理

#### 场景说明

当贷款出现逾期、欠息、经营恶化、担保弱化、资金挪用等情况时，银行需要进入预警管理或问题授信管理流程。

#### 重点核查内容

* 逾期本金和利息情况；
* 客户实际经营状态；
* 是否停产、停业或失联；
* 还款意愿是否下降；
* 保证人是否具备代偿能力；
* 抵押物是否可处置；
* 是否需要诉讼、保全、催收或重组；
* 是否需要调整风险分类。

#### 智能体应用价值

智能体可将分散风险信号进行归因和分级，形成：

```text
风险事件
触发原因
证据链
影响分析
处置建议
跟踪任务
```

## 7. 存续期业务中的典型风险信号

在后续尽职调查中，以下信号通常具有较高风险提示价值。

| 风险类别   | 典型信号                          |
| ------ | ----------------------------- |
| 资金用途风险 | 贷款资金流入关联方、个人账户、非经营领域，或资金回流借款人 |
| 经营恶化风险 | 收入下降、订单减少、纳税减少、账户流水下降、停工停产    |
| 现金流风险  | 回款变慢、应收账款增加、经营现金流为负、快进快出交易增多  |
| 融资风险   | 他行贷款增加、频繁新增融资、出现逾期欠息、依赖借新还旧   |
| 担保风险   | 抵押物贬值、查封冻结、重复抵押、保证人代偿能力下降     |
| 司法风险   | 新增诉讼、被执行、失信、资产保全、重大仲裁         |
| 管理风险   | 实控人变更、高管变动、股权纠纷、关联企业风险传导      |
| 合同违约风险 | 未按用途用款、未提供资料、突破财务约束、违反授信条件    |
| 舆情风险   | 负面新闻、监管处罚、质量事故、安全事故、环保事件      |
| 行业风险   | 行业下行、政策收紧、原材料波动、供应链中断         |

## 8. 存续期数据探针智能体的项目价值

### 8.1 从“人工检查”转向“智能探针”

传统贷后检查高度依赖客户经理人工查看资料、人工比对数据、人工撰写报告，存在效率低、信息滞后、标准不一、证据分散等问题。

存续期数据探针智能体可作为嵌入信贷管理流程中的智能分析工具，对客户存续期数据进行持续扫描、异常识别和证据归集。

### 8.2 从“单点数据查看”转向“多源证据链分析”

单一数据往往难以判断真实风险。例如，企业财务报表显示收入稳定，但账户流水下降、纳税减少、司法案件增加，这些信息结合起来才可能揭示真实风险。

智能体应具备多源数据融合能力，将不同系统中的数据串联为风险证据链。

### 8.3 从“事后发现”转向“前置预警”

传统贷后管理往往在逾期或欠息后才发现风险。智能体可以通过前置信号识别潜在风险，例如回款下降、资金异常流向、工商变更、他行融资增加、司法风险暴露等。

### 8.4 从“经验判断”转向“结构化风险研判”

智能体可以将客户经理经验、监管要求、内部制度和历史案例转化为结构化规则、指标体系和智能分析模板，提高风险判断的一致性。

## 9. 智能体可支持的核心功能模块

### 9.1 客户全景画像

整合客户基本信息、股权结构、授信信息、贷款余额、还款情况、账户流水、财务数据、担保信息、司法舆情等，形成客户存续期全景视图。

### 9.2 资金用途核查

自动追踪贷款资金流向，识别资金是否进入约定交易对手、是否转入关联方、是否存在回流、拆分转账、异常支付等情况。

### 9.3 经营变化监测

通过账户流水、财务报表、税务数据、订单信息、外部公开信息等，识别企业经营是否发生重大变化。

### 9.4 担保状态监控

持续监测抵押物估值、查封冻结、权属变化、保证人风险、担保覆盖率等，识别第二还款来源弱化风险。

### 9.5 外部风险扫描

自动采集或接入工商、司法、行政处罚、舆情、行业政策等外部风险信息，识别重大事项。

### 9.6 风险预警生成

基于规则、模型或大模型分析，对风险信号进行分级，形成预警结果和触发依据。

### 9.7 后续尽调报告生成

根据客户存续期数据和风险信号，自动生成贷后检查报告、后续尽调报告、预警处置建议、到期续贷评估报告等。

### 9.8 处置建议辅助

根据风险等级和风险类型，辅助输出差异化处置建议，例如：

* 继续观察；
* 补充资料；
* 现场检查；
* 追加担保；
* 压降额度；
* 暂停提款；
* 提前收贷；
* 调整风险分类；
* 移交风险处置。

存续期数据探针智能体可输出以下业务成果。

### 10.1 存续期风险扫描摘要

```text
客户名称：XXX有限公司
贷款余额：人民币 3,000 万元
贷款用途：采购原材料
本期风险等级：中风险
主要风险信号：

1. 开展专项资金用途核查；
2. 要求客户补充交易合同、发票和物流资料；
3. 关注关联企业资金占用情况；
```

### 10.2 资金用途核查结论

```text
经比对贷款审批用途、合同约定、放款流水及交易凭证，本笔贷款部分资金流向与约定采购用途存在不一致。系统识别到放款后三日内有 500 万元转入借款人关联企业账户，交易摘要为“往来款”，未见对应采购合同及发票。建议客户经理进一步核实交易真实性和资金最终用途。
```

```text
该客户当前经营仍在持续，但近期账户回款下降明显，纳税规模同步下降，且他行融资余额增加。虽然尚未出现逾期，但第一还款来源稳定性有所弱化。建议续贷时适当压降额度，并要求补充有效担保，同时提高贷后检查频率。
```

## 11. 产品建设中的业务边界

存续期数据探针智能体应定位为银行存续期管理中的智能辅助工具，而不是完全替代客户经理、审批人员或风险管理人员的决策系统。

其业务边界包括：

1. 智能体可以识别风险线索，但最终风险判断仍需业务人员确认；
2. 智能体可以生成尽调报告初稿，但关键事实和结论需人工复核；
3. 智能体可以提出处置建议，但授信调整、提前收贷、风险分类调整等需遵循银行内部审批流程；
4. 智能体可以辅助解释数据异常，但不能在证据不足时直接认定客户违约；
5. 智能体需要严格遵守数据安全、客户隐私、模型合规和银行内部权限管理要求。

* 趋势分析；

* 贷后检查报告归档。

## 13. 一句话总结

银行贷款后续尽职调查中的存续期业务，是银行在贷款未结清期间，对借款人经营、财务、资金用途、还款来源、担保状态、合同履约和风险变化进行持续跟踪、核查、预警和处置的全过程。

“存续期数据探针智能体”的核心价值在于：

> 通过多源数据整合、智能风险识别、证据链分析和报告自动生成，帮助银行从被动贷后检查转向主动风险探测，从经验式判断转向数据驱动的存续期风险管理。
