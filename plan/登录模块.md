## plan

你是一名资深 Agent 开发工程师 / 全栈架构师。

我正在开发一个【存续期业务数据探针智能体】。

请使用 Plan 模式工作：先进行模块分析和方案设计，不要直接写代码。

【模块背景】
模块用于：
- 【说明模块的业务用途】
“存续期数据探针智能体”用户登录模块，仅支持内部注册用户登录使用。

- 【说明目标用户或使用场景】
目标用户为银行贷后管理人员、尽职调查人员、数据分析人员、存续期系统管理员等。

智能体用户登录认证场景使用。

- 【说明当前已有能力或现有限制】

无。

【核心目标】
本模块需要实现：
1. 支持用户两种方式登录，账号/邮箱+密码登录，或使用企业SSO登录；
2. 支持用户找回密码、修改密码等功能；
3. 支持用户登录状态管理，登录态过期时自动跳转到登录页面，登录态有效时可直接访问系统功能；
4. 支持用户权限管理，包括不同用户类型的权限控制，如普通用户、管理员等；
5. 支持登录日志记录，包括登录时间、登录IP、登录方式等信息，便于后续审计和追踪。
6. 支持登录认证连续错误次数限制，超过限制则锁定账号，需管理员解锁或通过找回密码功能解锁。
7. 支持登录认证异常处理，如输入缺失、账号不存在、密码错误、SSO登录失败等情况，返回明确的错误提示信息。
8. 支持登录认证安全防护，如防止暴力破解、SQL注入、CSRF攻击等安全措施，确保用户账号和系统数据的安全性。

【输入信息】
系统可能接收以下输入：
- 用户输入账号+密码;
- 用户输入邮箱+密码;
- 企业SSO登录信息;

【期望输出】
系统最终应输出：
- 【登录认证状态，成功/失败】
登录认证成功则进入智能体主界面，登录认证失败则提示错误信息并停留在登录页面。
- 【登录认证连续错误次数限制】
登录认证连续错误次数超过限制则锁定账号，需管理员解锁或通过找回密码功能解锁。

【标准】
请按照以下标准设计方案：
1. 输出必须可被用户引用作为判断依据，禁止输出定义不明确、模糊不清的分析内容；
2. 模块职责必须清晰；
3. 数据流必须可追踪；
4. 异常处理必须明确；
5. 方案应适合工程落地，而不是只停留在概念描述。

【边界】
设计时必须遵守：
1. 技术栈限定为：【例如 TypeScript + Node.js + Electron + FastAPI】；

- 界面开发框架：`React`、`astryx`，登录模块选择`astryx`模版`login-sso`;
- 服务端开发语言：`Nodejs`；
- 服务端开发框架：`Express`、`FastAPI`等；;
2. 不允许使用：【不允许的库 / 服务 / 架构】；
