# 存续期数据探针智能体｜配置数据源模块

你现在是一个资深全栈架构师、数据平台架构师、数据库性能优化专家和银行数据安全架构师。请使用 **plan 模式**，先不要直接编写代码，而是围绕项目 **“存续期数据探针智能体”** 的 **“配置数据源模块”** 完成一份技术架构设计方案。

重点要求：该模块需要支持数据库连接配置、数据库 Compass 工具、元数据同步、表结构浏览、表关系推断、CSV 导入补充数据源，并且需要支持读取大表。但在读取大表时，必须充分考虑耗时、权限管控、数据安全、连接池、慢查询、数据库压力、拦截、限流、超时、熔断等风险，避免分析工具影响线上业务系统访问数据。

## 一、项目背景

项目名称：**存续期数据探针智能体**

项目面向银行贷款后续尽职调查、贷后管理和存续期风险监测场景，需要接入多类业务数据源，为后续 AI Agent 提供数据探索、SQL 生成、风险识别、证据链生成和报告生成能力。

当前模块为：**配置数据源模块**。

模块能力包括：

1. 配置数据库连接；
2. 测试数据库连接；
3. 同步数据库元数据；
4. 构建 Database Compass / Schema Explorer；
5. 查看表列表、字段、索引、外键、表结构、表关系；
6. 查看样例数据；
7. 支持读取大表；
8. 支持手动导入表数据或 CSV 文件；
9. 为 AI Agent 提供结构化数据上下文；
10. 对数据库访问进行权限、安全、性能和审计管控。

## 十一、限流、降级和熔断设计

请重点设计大表访问和生产库访问下的限流、降级、熔断机制。

### 1. 限流

至少包括：

* 单用户 QPS；
* 单数据源 QPS；
* 单租户 QPS；
* 单表访问频率；
* 单用户大表查询并发数；
* 单数据源大表查询并发数；
* 全局查询并发数；
* 单时间窗口内慢查询次数。

### 2. 降级

当数据库压力较高或查询变慢时，应支持：

* 禁止样例数据读取，仅允许查看缓存元数据；
* 降低预览行数；
* 降低最大字段数；
* 禁止大表读取；
* 禁止复杂表关系推断；
* 禁止 AI Agent 自动查询；
* 使用上次缓存结果；
* 提示用户稍后重试。

### 3. 熔断

触发条件可包括：

* 连续连接失败；
* 查询超时次数超过阈值；
* 慢查询比例超过阈值；
* 数据库返回资源不足；
* 连接池耗尽；
* 单数据源错误率过高；
* 生产库响应时间超过阈值；
* 被管理员手动熔断。

熔断后行为：

* 暂停该数据源新查询；
* 保留元数据缓存浏览；
* 禁止样例数据读取；
* 禁止 AI Agent 查询；
* 记录审计日志；
* 通知管理员；
* 支持自动半开恢复；
* 支持手动恢复。

请给出状态机：

```text
Closed → Open → Half-Open → Closed
```

并用 Mermaid 画出熔断状态流转图。
