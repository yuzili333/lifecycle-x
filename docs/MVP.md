# 存续期数据探针智能体 MVP 定义

## 1. MVP 目标

存续期数据探针智能体 MVP 面向银行贷后管理人员、后续尽职调查人员和风险管理人员，优先验证“业务人员通过自然语言完成存续期数据探索、风险线索识别、证据链整理和报告初稿生成”的核心价值。

MVP 不追求一次性覆盖所有贷后管理场景，而是先聚焦高频、高价值、数据结构相对明确的存续期风险核查场景，验证以下假设：

1. 业务人员无需编写 SQL 或 Python，即可发起数据分析任务。
2. 系统能够从导入数据或连接数据源中识别关键字段和分析口径。
3. 系统输出的风险信号必须带有证据链，能够被业务人员引用和复核。
4. 系统能够支持多轮追问和局部调整，而不是只输出一次性固定报表。
5. 系统能够生成可归档的 Markdown / PDF / JSON 结构化结果。

## 2. MVP 用户与场景

### 2.1 目标用户

| 用户类型 | 主要诉求 | MVP 覆盖程度 |
|---|---|---|
| 贷后管理人员 | 快速识别客户经营、回款、资金用途和担保变化 | 覆盖核心分析和报告初稿 |
| 后续尽职调查人员 | 归集异常证据、形成尽调线索和核查建议 | 覆盖证据链和建议动作 |
| 风险管理人员 | 查看预警依据、风险等级和处置建议 | 覆盖结构化风险信号 |
| 数据分析/科技人员 | 降低临时取数和专项分析支持压力 | 覆盖 CSV/数据库接入的基础链路 |
| 审核/管理人员 | 复核报告依据与过程 trace | 覆盖归档和执行 trace 的基础能力 |

### 2.2 首期业务场景

MVP 首期只覆盖以下四类场景：

1. 放款后资金用途核查：识别贷款资金是否流入关联方、个人账户、非经营交易对手或存在回流迹象。
2. 回款账户变化分析：识别近周期回款下降、快进快出、大额异常支出等现金流风险。
3. 外部风险线索整理：识别司法诉讼、被执行、行政处罚、负面舆情等风险变化。
4. 后续尽调报告初稿生成：根据风险信号、证据链和分析口径生成报告初稿。

## 3. MVP 范围

### 3.1 必须包含

| 模块 | MVP 能力 |
|---|---|
| 自然语言任务入口 | 支持输入分析目标、客户名称、时间范围、输出格式等自然语言需求 |
| 数据源导入 | 支持 CSV 文件导入；数据库连接作为可选验证能力 |
| 数据结构识别 | 识别字段名称、字段类型、金额字段、日期字段、客户字段、交易对手字段 |
| 分析计划生成 | 将用户意图拆解为数据选择、统计口径、风险规则和输出物 |
| 风险规则识别 | 覆盖资金用途风险、回款下降、司法风险、担保弱化的基础规则 |
| 证据链生成 | 每条结论必须关联证据编号、数据来源、记录范围和统计口径 |
| 多轮追问 | 支持基于上一轮结果调整时间范围、筛选客户、拆分维度、局部纠错 |
| 报告输出 | 支持 Markdown、JSON、PDF 报告初稿 |
| 归档引用 | 支持保存分析结果、报告、证据链和执行 trace |
| 异常处理 | 输入缺失、数据源失败、字段缺失、结果冲突、模型输出非法时返回结构化错误 |

### 3.2 暂不包含

| 暂不包含能力 | 说明 |
|---|---|
| 直接替代风险分类决策 | MVP 只给出风险线索和建议动作，不做最终风险分类认定 |
| 自动发起审批或处置流程 | 不直接触发授信调整、提前收贷、风险处置等正式流程 |
| 全量外部数据实时采集 | 工商、司法、舆情等外部接口首期可通过导入数据验证 |
| 复杂机器学习模型训练 | 首期以规则、统计口径和可解释分析为主 |
| 多机构权限体系 | 首期只验证单机构或单租户使用流程 |
| 移动端 | 首期只覆盖桌面端操作台 |

## 4. MVP 核心用户流程

1. 业务人员选择或导入数据源。
2. 系统展示数据源字段、样例数据、识别出的客户字段、金额字段、日期字段。
3. 业务人员输入任务，例如：“核查 XX 公司近三个月贷款资金是否流入关联方，并生成尽调摘要。”
4. 系统生成分析计划，列出将使用的数据源、筛选条件、统计口径和风险规则。
5. 业务人员确认或调整计划。
6. 系统执行分析并输出风险信号、关键指标、证据链、图表和报告初稿。
7. 业务人员继续追问，例如“把时间范围扩展到半年”“按交易对手拆分”“剔除工资类交易”。
8. 系统基于上下文重新分析并保留版本记录。
9. 业务人员导出报告并归档。

## 5. MVP 成功标准

| 指标 | 验收标准 |
|---|---|
| 任务完成效率 | 典型资金用途核查任务从导入数据到报告初稿不超过 10 分钟 |
| 证据可追溯 | 100% 风险信号包含证据编号和数据来源 |
| 口径明确性 | 100% 指标包含统计口径、时间范围和筛选条件 |
| 异常可解释 | 字段缺失、数据源失败、结果冲突均返回结构化原因和补救建议 |
| 报告可用性 | 业务人员可基于报告初稿进行人工修订和归档 |
| 多轮可用性 | 至少支持 3 轮连续追问，且能引用上一轮结果 |

## 6. 首期样例数据要求

| 数据对象 | 关键字段 |
|---|---|
| 客户信息 | 客户编号、客户名称、统一社会信用代码、行业、实际控制人、关联企业 |
| 授信合同 | 贷款编号、贷款金额、贷款用途、放款日期、到期日期、还款方式、合同约束 |
| 账户流水 | 客户编号、账户号、交易日期、交易金额、借贷方向、交易对手、交易摘要、交易后余额 |
| 交易对手关系 | 交易对手名称、关系类型、是否关联方、是否个人账户、证据来源 |
| 外部风险 | 客户编号、风险类型、发生日期、事项描述、来源、影响程度 |
| 担保信息 | 押品编号、押品类型、估值、登记状态、查封冻结状态、担保覆盖率 |

## 7. MVP 风险与约束

1. 数据质量不足会影响分析结果，系统必须提示字段缺失和口径限制。
2. 大模型可能生成不可靠表述，报告生成必须只引用结构化分析结果和证据链。
3. 银行业务规则存在机构差异，阈值和风险等级规则需配置化。
4. 涉及客户、账户、资金流水等敏感数据，必须按银行内部数据安全规范处理。
5. MVP 输出只能作为辅助判断，最终风险认定仍需业务人员复核。

## 8. MVP 交付物

1. 桌面端业务工作台原型。
2. 数据源导入与 schema 识别能力。
3. 自然语言任务到分析计划的转换能力。
4. 首批风险规则和指标口径。
5. 结构化分析结果、证据链、trace 和报告初稿。
6. 业务验收样例数据集。
7. MVP 试点反馈表和问题清单。
