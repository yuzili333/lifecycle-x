# 存续期数据探针智能体业务需求说明书（BRD）

## 1. 功能名称

存续期数据探针智能体

## 2. 功能描述

存续期数据探针智能体用于银行贷款发放后至结清、额度终止或风险处置完成前的存续期管理场景，辅助贷后管理人员、后续尽职调查人员和风险管理人员完成客户经营变化监测、资金用途核查、担保状态跟踪、风险线索识别、证据链整理和报告初稿生成。

本功能的核心目标是降低业务人员使用 SQL、Python 和专项研发取数的门槛，使业务人员能够通过自然语言描述分析目标，完成数据查询、清洗、统计、异常识别、风险说明和报告归档。系统输出应明确展示数据来源、统计口径、风险触发条件和证据编号，确保业务人员可以复核和引用。

本功能定位为存续期管理辅助工具，不替代客户经理、审批人员或风险管理人员的最终判断，不直接触发授信调整、提前收贷、风险分类调整或风险处置流程。

## 3. 相关用户

| 用户类型 | 业务职责 | 关注点 |
|---|---|---|
| 贷后管理人员 | 定期检查客户经营、还款、资金用途和担保状态 | 快速识别异常、形成检查记录 |
| 后续尽职调查人员 | 对重大事项或风险客户开展专项核查 | 证据链完整、核查建议明确 |
| 风险管理人员 | 跟踪预警客户和问题贷款 | 风险类别、触发依据、处置建议 |
| 业务管理人员 | 查看辖内客户风险变化和检查质量 | 管理汇总、归档结果、审计追踪 |
| 数据分析/科技人员 | 提供数据源、字段口径和系统支持 | 降低临时取数压力，保证数据安全 |
| 合规/审计人员 | 检查系统输出依据和操作留痕 | trace 完整、权限可控、结论可复核 |

## 4. 业务规则

### 4.1 接入分析数据

- 导入数据源：业务人员可导入 CSV 文件或选择已配置的数据源；如数据源不存在或无权限访问，则终止分析并提示补充数据源或申请权限。
- 识别数据结构：根据字段名称、字段类型和样例值识别客户、贷款、账户、日期、金额、交易对手、担保、外部风险等字段；如关键字段无法识别，则提示业务人员手动映射。
- 校验数据完整性：对客户编号、客户名称、贷款编号、交易日期、交易金额等关键字段进行完整性检查；如缺失影响分析结论，则标记为数据缺口。
- 记录数据来源：每个数据源应生成来源编号，并记录导入时间、数据范围、字段映射和数据质量摘要。

### 4.2 发起自然语言分析任务

- 识别分析意图：根据用户输入识别任务类型，包括资金用途核查、回款变化分析、经营变化分析、担保状态监控、外部风险扫描、报告生成等。
- 提取分析条件：从用户输入中提取客户名称、客户编号、贷款编号、时间范围、输出格式、对比周期等条件；如关键条件缺失，则提示用户补充。
- 生成分析计划：将任务拆解为数据源选择、筛选条件、统计口径、风险规则、证据链输出和报告输出步骤；分析计划应在执行前可被查看。
- 支持多轮调整：用户可基于上一轮结果继续调整时间范围、筛选条件、统计维度和图表类型；系统应保留上下文和版本记录。

### 4.3 识别风险信号

- 核查资金用途：比对贷款用途、放款流水、交易对手和交易摘要；如资金流入关联方、个人账户、非经营领域或疑似回流，应生成资金用途风险线索。
- 分析回款变化：按时间周期统计回款金额、回款频率、主要交易对手和资金留存时间；如出现明显下降、快进快出或异常大额转出，应生成现金流风险线索。
- 监控经营变化：结合财务、流水、税务、订单等数据识别收入下降、纳税减少、经营现金流弱化等变化；如数据不足，应标记为待补充。
- 跟踪担保状态：识别押品估值、登记状态、查封冻结状态和担保覆盖率变化；如押品贬值或权属受限，应生成担保弱化风险线索。
- 扫描外部风险：识别司法诉讼、被执行、行政处罚、负面舆情、工商变更等事项；如事项发生在贷款存续期内，应关联客户和贷款。
- 输出风险等级：风险等级应基于命中规则、影响金额、发生时间、证据数量和业务阈值生成；阈值未确认时应标记为待业务确认。

### 4.4 生成证据链

- 生成证据编号：每条风险信号至少关联一条证据，证据包含来源编号、查询编号、记录编号、时间范围和统计口径。
- 绑定原始数据：证据应能追溯到导入文件、数据库表或历史归档结果；如原始记录不可定位，则不得作为正式证据输出。
- 说明统计口径：指标类证据必须说明计算公式、筛选条件、对比周期和排除项。
- 处理多源冲突：当多个数据源对同一事实给出不同结果时，不自动覆盖，应输出冲突项、来源差异和复核建议。

### 4.5 生成报告与归档

- 生成报告初稿：根据分析结果生成贷后检查摘要、后续尽调报告、风险预警摘要或到期续贷评估初稿。
- 限制报告结论：报告只能引用结构化分析结果和证据链，不得新增无证据支持的事实判断。
- 输出多种格式：支持 Markdown、JSON、PDF 和可视化图片卡片；如输出失败，应保留结构化结果并提示失败原因。
- 归档分析结果：归档内容包括任务输入、分析计划、执行结果、风险信号、证据链、报告文件和 trace。

### 4.6 异常与人工复核

- 输入缺失处理：客户、时间范围、数据源或关键字段缺失时，应返回明确补充项。
- 数据源失败处理：连接失败或文件解析失败时，应记录失败原因；如存在替代数据源，可降级分析并标记结果不完整。
- 输出异常处理：模型输出不符合结构化协议时，应重试或返回结构化错误，不得展示无法校验的结论。
- 人工复核要求：高风险结论、冲突结论、证据不足结论均应提示业务人员复核。

## 7. 其他附件与补充说明

待业务方确认事项：

1. 首期试点机构、试点客户范围和业务条线。
2. 数据源清单、字段字典、样例数据和权限边界。
3. 资金用途异常、回款下降、担保弱化等风险规则阈值。
4. 报告模板是否需匹配现有贷后检查或尽调报告格式。
5. 证据链能否引用原始流水明细，是否需要脱敏展示。
6. 风险等级与银行内部预警等级之间的映射关系。
7. 历史分析结果和报告归档的保存期限。
8. 审计、合规、数据安全和模型合规要求。
