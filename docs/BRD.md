# 存续期数据探针智能体业务需求说明书（BRD）

## 1. 功能名称

存续期数据探针智能体

## 2. 功能描述

存续期数据探针智能体用于银行贷款发放后至结清、额度终止或风险处置完成前的存续期管理场景，辅助贷后管理人员、后续尽职调查人员和业务分析人员完成数据提取、数据清洗、数据加工、数据分析、分析依据整理和报告初稿生成。

本功能的核心目标是降低业务人员使用 SQL、Python 和专项研发取数的门槛，使业务人员能够通过自然语言描述分析目标，完成数据查询、字段清洗、数据关联、分类汇总、趋势对比、明细筛选和报告归档。系统输出应明确展示数据来源、加工步骤、筛选条件和统计口径，确保业务人员可以复核和引用。

本功能定位为存续期数据分析辅助工具，不由智能体直接输出风险信号、风险等级、预警结论或处置建议，不替代客户经理、审批人员或风险管理人员的最终判断，不直接触发授信调整、提前收贷、风险分类调整或风险处置流程。

## 3. 相关用户

| 用户类型 | 业务职责 | 关注点 |
|---|---|---|
| 贷后管理人员 | 定期整理客户经营、还款、资金用途和担保相关数据 | 快速获得可复核的数据明细、汇总指标和检查材料 |
| 后续尽职调查人员 | 对客户、贷款、账户流水和外部事项开展专项数据核查 | 数据来源清晰、口径明确、材料可引用 |
| 风险管理人员 | 查看业务人员提交的数据分析依据 | 数据范围、加工过程和统计口径透明 |
| 业务管理人员 | 查看辖内客户数据分析结果和检查材料归档情况 | 管理汇总、归档结果、审计追踪 |
| 数据分析/科技人员 | 提供数据源、字段口径和系统支持 | 降低临时取数压力，保证数据安全 |
| 合规/审计人员 | 检查系统输出依据和操作留痕 | trace 完整、权限可控、结论可复核 |

## 4. 业务规则

### 4.1 接入分析数据

- 导入数据源：业务人员可导入 CSV 文件或选择已配置的数据源；如数据源不存在或无权限访问，则终止分析并提示补充数据源或申请权限。
- 识别数据结构：根据字段名称、字段类型和样例值识别客户、贷款、账户、日期、金额、交易对手、担保、外部事项等字段；如关键字段无法识别，则提示业务人员手动映射。
- 校验数据完整性：对客户编号、客户名称、贷款编号、交易日期、交易金额等关键字段进行完整性检查；如缺失影响分析结论，则标记为数据缺口。
- 记录数据来源：每个数据源应生成来源编号，并记录导入时间、数据范围、字段映射和数据质量摘要。

### 4.2 发起自然语言分析任务

- 识别分析意图：根据用户输入识别任务类型，包括资金流向整理、回款变化分析、经营数据对比、担保数据汇总、外部事项整理、报告生成等。
- 提取分析条件：从用户输入中提取客户名称、客户编号、贷款编号、时间范围、输出格式、对比周期等条件；如关键条件缺失，则提示用户补充。
- 生成分析计划：将任务拆解为数据源选择、字段映射、清洗规则、筛选条件、加工步骤、统计口径、数据血缘输出和报告输出步骤；分析计划应在执行前可被查看。
- 支持多轮调整：用户可基于上一轮结果继续调整时间范围、筛选条件、统计维度和图表类型；系统应保留上下文和版本记录。

### 4.3 执行数据加工与分析

- 提取资金流向数据：根据贷款、账户、交易日期、交易金额、交易对手和交易摘要提取放款后资金流水，形成可筛选明细。
- 加工回款变化指标：按日、月、季度等周期统计回款金额、回款频率、主要交易对手和资金留存时间；系统只输出事实性指标，不判断风险等级。
- 对比经营数据变化：结合财务、流水、税务、订单等数据进行收入、纳税、现金流、订单等指标的周期对比；如数据不足，应标记为待补充。
- 汇总担保相关数据：整理押品估值、登记状态、查封冻结状态和担保覆盖率等数据，输出变化明细和汇总结果。
- 整理外部事项数据：对司法诉讼、被执行、行政处罚、负面舆情、工商变更等事项进行结构化归集和时间线展示；是否构成风险由业务人员判断。
- 输出分析结果：分析结果应包含明细数据、汇总指标、筛选条件、统计口径和数据来源，不输出风险信号、风险等级或处置建议。

### 4.4 生成数据血缘

- 生成结果编号：每个明细结果、汇总指标或报告片段应关联结果编号，结果编号包含来源编号、查询编号、记录范围、加工步骤、时间范围和统计口径。
- 绑定原始数据：分析结果应能追溯到导入文件、数据库表或历史归档结果；如原始记录不可定位，则不得作为可复核依据输出。
- 说明统计口径：指标类结果必须说明计算公式、筛选条件、对比周期和排除项。
- 处理多源冲突：当多个数据源对同一事实给出不同结果时，不自动覆盖，应输出冲突项、来源差异和复核建议。

### 4.5 生成报告与归档

- 生成报告初稿：根据分析结果生成贷后检查数据摘要、后续尽调材料、客户经营数据分析或到期续贷数据分析初稿。
- 限制报告结论：报告只能引用结构化分析结果和数据血缘，不得新增无数据支持的事实判断，不输出风险定性结论。
- 输出多种格式：支持 Markdown、JSON、PDF 和可视化图片卡片；如输出失败，应保留结构化结果并提示失败原因。
- 归档分析结果：归档内容包括任务输入、分析计划、执行结果、数据血缘、报告文件和 trace。

### 4.6 异常与人工复核

- 输入缺失处理：客户、时间范围、数据源或关键字段缺失时，应返回明确补充项。
- 数据源失败处理：连接失败或文件解析失败时，应记录失败原因；如存在替代数据源，可降级分析并标记结果不完整。
- 输出异常处理：模型输出不符合结构化协议时，应重试或返回结构化错误，不得展示无法校验的结论。
- 人工复核要求：多源冲突、数据缺口、口径不明确或原始记录不可定位的结果均应提示业务人员复核。

## 7. 其他附件与补充说明

待业务方确认事项：

1. 首期试点机构、试点客户范围和业务条线。
2. 数据源清单、字段字典、样例数据和权限边界。
3. 资金流向、回款统计、担保数据和外部事项的字段映射与统计口径。
4. 报告模板是否需匹配现有贷后检查或尽调报告格式。
5. 数据血缘能否引用原始流水明细，是否需要脱敏展示。
6. 报告中哪些内容属于数据事实，哪些内容必须由业务人员人工判断。
7. 历史分析结果和报告归档的保存期限。
8. 审计、合规、数据安全和模型合规要求。
