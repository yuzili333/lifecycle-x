# 存续期数据探针智能体产品需求文档（PRD）

## 1. 产品概述

存续期数据探针智能体是一款面向银行贷后管理与后续尽职调查场景的桌面端业务分析工具。产品通过自然语言交互、多源数据接入、风险信号识别、证据链生成和报告输出，帮助业务人员从被动查看固定报表转向主动探索风险线索。

产品首期以 MVP 验证为目标，优先支持 CSV 导入和可配置数据源，围绕资金用途核查、回款变化分析、外部风险扫描和尽调报告初稿生成形成闭环。

## 2. 产品目标

1. 降低业务人员开展数据分析的技术门槛。
2. 提升贷后检查和后续尽调的分析效率。
3. 将风险结论与证据链绑定，提升可复核性。
4. 支持多轮探索式分析，适配业务人员动态调整思路的工作方式。
5. 形成可归档、可引用、可审计的分析资产。

## 3. 用户画像

| 用户 | 使用频率 | 核心任务 | 痛点 |
|---|---|---|---|
| 客户经理/贷后经理 | 高频 | 日常贷后检查、客户经营变化监测 | 数据分散，手工比对慢 |
| 后续尽调人员 | 中高频 | 对预警客户开展专项核查 | 证据分散，报告撰写耗时 |
| 风险经理 | 中频 | 查看风险信号和处置建议 | 预警依据不透明 |
| 业务管理人员 | 中频 | 监督检查质量和归档情况 | 缺少过程留痕 |
| 科技/数据人员 | 低频 | 提供数据接入和字段口径支持 | 临时取数需求多 |

## 4. 产品范围

### 4.1 首期范围

- 桌面端工作台。
- 自然语言任务输入。
- CSV 数据导入。
- 数据源字段识别和字段映射。
- 分析计划展示。
- 资金用途、回款下降、司法风险、担保弱化等基础风险识别。
- 关键指标、风险信号、证据链、建议动作输出。
- Markdown、JSON、PDF 报告初稿生成。
- 历史结果归档和引用。
- 结构化异常返回和执行 trace。

### 4.2 非首期范围

- 直接写入银行核心业务系统。
- 自动审批、自动预警下发、自动风险分类调整。
- 全量实时外部数据采集。
- 跨机构多租户复杂权限。
- 移动端应用。
- 复杂模型训练平台。

## 5. 信息架构

桌面端建议包含以下一级导航：

| 一级导航 | 功能说明 |
|---|---|
| 工作台 | 发起自然语言任务、查看分析计划、结果和证据链 |
| 数据源 | 管理 CSV、数据库连接、字段映射和数据质量 |
| 证据库 | 查看证据编号、来源、记录、口径和引用关系 |
| 报告库 | 查看和导出历史 Markdown、PDF、JSON 报告 |
| 审计追踪 | 查看任务执行 trace、重试、降级、冲突和错误 |
| 设置 | 配置规则阈值、输出模板、权限和安全策略 |

## 6. 核心功能需求

### 6.1 数据源管理

#### 功能说明

业务人员或数据管理员可导入 CSV 文件，或选择已配置的数据库连接。系统读取字段、样例数据和数据质量摘要，并提示关键字段映射情况。

#### 主要交互

1. 上传 CSV 文件。
2. 查看字段识别结果。
3. 修正字段映射。
4. 保存数据源。
5. 查看数据质量提示。

#### 验收标准

- 能识别文本、日期、金额、枚举、编号等常见字段类型。
- 能提示客户字段、贷款字段、金额字段、日期字段、交易对手字段是否缺失。
- 字段缺失时给出明确补充建议。
- 数据源应生成唯一来源编号。

### 6.2 自然语言任务入口

#### 功能说明

用户通过自然语言描述分析目标，系统识别任务类型、客户对象、时间范围、输出格式和需要引用的历史结果。

#### 示例输入

- “核查 XX 公司放款后三个月资金是否流入关联方，并输出证据链。”
- “分析该客户近半年回款是否下降，按主要交易对手拆分。”
- “基于本次风险信号生成后续尽调报告初稿。”

#### 验收标准

- 能识别资金用途核查、回款分析、外部风险扫描、报告生成等任务类型。
- 关键条件缺失时不直接执行，应要求用户补充。
- 能基于上一轮分析结果继续追问。

### 6.3 分析计划生成

#### 功能说明

系统在执行前展示分析计划，包含数据源、筛选条件、统计口径、风险规则和输出物。

#### 计划内容

| 内容 | 说明 |
|---|---|
| 任务类型 | 资金用途核查、回款分析等 |
| 数据来源 | 参与分析的数据源编号 |
| 筛选条件 | 客户、贷款、时间范围等 |
| 统计口径 | 计算公式、对比周期、排除规则 |
| 风险规则 | 将执行的风险识别规则 |
| 输出物 | JSON、Markdown、PDF、图表 |

#### 验收标准

- 执行前可查看计划。
- 计划中不得出现无法解释的数据来源或口径。
- 用户可调整时间范围、客户范围和输出格式。

### 6.4 风险信号识别

#### 功能说明

系统根据结构化数据和规则识别风险线索。首期风险规则以可解释规则为主。

#### 首批规则

| 风险类别 | 触发示例 | 输出要求 |
|---|---|---|
| 资金用途风险 | 放款资金流入关联方、个人账户、非经营摘要、短期回流 | 风险原因、涉及金额、交易对手、证据编号 |
| 回款下降风险 | 本期回款较上期明显下降 | 下降比例、对比周期、账户范围、证据编号 |
| 现金流异常 | 快进快出、大额异常转出、频繁个人账户转账 | 交易明细、金额、频率、证据编号 |
| 司法风险 | 新增诉讼、被执行、失信、重大仲裁 | 事项类型、发生日期、来源、证据编号 |
| 担保弱化风险 | 押品贬值、查封冻结、担保覆盖率下降 | 押品信息、变化情况、证据编号 |

#### 验收标准

- 每条风险信号必须包含风险类别、等级、原因、证据编号和建议动作。
- 无证据不得输出确定性风险结论。
- 阈值未确认时，应标记“待业务确认阈值”。

### 6.5 证据链管理

#### 功能说明

证据链用于支撑风险信号和报告结论，确保结论可复核、可引用。

#### 证据字段

| 字段 | 说明 |
|---|---|
| 证据编号 | 唯一标识证据 |
| 数据来源编号 | 指向 CSV、数据库表或历史 artifact |
| 查询编号 | 指向本次分析中的查询或统计步骤 |
| 原始记录范围 | 指向原始记录或记录集合 |
| 时间范围 | 证据覆盖的日期范围 |
| 统计口径 | 指标计算方式和筛选条件 |
| 证据摘要 | 面向业务人员的事实描述 |

#### 验收标准

- 风险信号、指标和报告结论都能引用证据编号。
- 证据摘要不能脱离原始数据产生新事实。
- 多源冲突应形成冲突 trace。

### 6.6 报告生成

#### 功能说明

系统基于结构化结果生成报告初稿，支持业务人员下载、修订和归档。

#### 报告类型

1. 存续期风险扫描摘要。
2. 资金用途核查结论。
3. 后续尽调报告初稿。
4. 到期续贷风险评估初稿。

#### 报告章节

- 客户与贷款概况。
- 分析范围和数据来源。
- 关键指标。
- 主要风险信号。
- 证据链摘要。
- 风险等级建议。
- 后续核查和处置建议。
- 数据缺口和人工复核事项。

#### 验收标准

- 报告中的事实必须能追溯到证据编号。
- 报告必须展示数据范围和统计口径。
- 数据不足时必须写明影响范围。

### 6.7 历史上下文与多轮分析

#### 功能说明

用户可基于历史分析继续追问、纠错和生成报告。

#### 示例

- “把上次结果中关联方交易按交易对手排序。”
- “剔除工资、税费和社保交易后重新计算。”
- “把这次分析结果追加到报告。”

#### 验收标准

- 系统能识别用户引用的历史任务或 artifact。
- 每次重算都生成新的版本记录。
- 局部纠错不得覆盖原始结果，应保留版本差异。

### 6.8 异常处理与审计追踪

#### 功能说明

系统应记录任务执行过程中的失败、重试、降级、冲突和模型输出校验结果。

#### 异常类型

| 异常 | 处理规则 |
|---|---|
| 输入缺失 | 返回缺失字段和补充建议 |
| 数据源失败 | 重试；失败后记录原因；可降级时标记部分成功 |
| 字段缺失 | 提示手动映射或补充数据 |
| 多源冲突 | 输出冲突项和复核建议 |
| 输出格式非法 | 重新校验或返回结构化错误 |
| 分析无法完成 | 返回失败阶段、已完成步骤和下一步建议 |

#### 验收标准

- 所有异常应有错误码、可恢复标识和 trace 编号。
- 降级结果必须标记为部分成功。
- 高风险但证据不足时输出风险线索，不输出最终认定。

## 7. 页面需求

### 7.1 工作台页面

核心区域：

- 自然语言任务输入框。
- 数据源选择器。
- 分析计划卡片。
- 风险信号列表。
- 关键指标区。
- 证据链侧栏。
- 报告导出按钮。
- 执行 trace 抽屉。

关键状态：

- 空状态：提示导入数据源或选择历史数据。
- 计划待确认：展示分析计划和可调整项。
- 执行中：展示当前步骤。
- 成功：展示指标、风险和证据。
- 部分成功：展示已完成结果和数据缺口。
- 失败：展示结构化错误和补救建议。

### 7.2 数据源页面

核心区域：

- 数据源列表。
- CSV 上传入口。
- 数据库连接配置入口。
- 字段识别结果。
- 字段映射编辑。
- 数据质量摘要。

### 7.3 证据库页面

核心区域：

- 证据编号列表。
- 证据来源和原始记录范围。
- 证据摘要。
- 被引用的风险信号和报告。
- 冲突证据提示。

### 7.4 报告库页面

核心区域：

- 报告列表。
- 报告类型、客户、生成时间、状态。
- Markdown / PDF / JSON 下载。
- 报告引用的证据链。

## 8. 数据与协议需求

### 8.1 任务请求

任务请求应包含任务编号、用户意图、数据源编号、上下文 artifact、客户或贷款约束、时间范围和输出格式。

### 8.2 分析结果

分析结果应包含任务状态、摘要、指标、风险信号、证据链、建议动作、artifact 和 trace。

### 8.3 错误返回

错误返回应包含错误码、错误说明、是否可恢复、是否可重试、是否使用降级方案和 trace 编号。

## 9. 权限与安全需求

1. 数据库账号、客户信息、账户流水、征信和司法数据属于敏感信息，应按银行内部标准加密、脱敏和审计。
2. 桌面端不得明文保存数据库密码。
3. 用户只能访问授权数据源。
4. 报告导出应记录导出人、导出时间、报告编号和数据范围。
5. 外部模型或服务使用前需确认数据出域和脱敏策略。

## 10. 非功能需求

| 类型 | 要求 |
|---|---|
| 性能 | MVP 典型 CSV 分析任务应在 10 分钟内完成，具体数据量上限待补充 |
| 可用性 | 关键错误应给出业务可理解的补救建议 |
| 可追溯 | 所有结论、指标和报告均需关联证据或 trace |
| 可配置 | 风险阈值、报告模板和字段映射应支持配置 |
| 可扩展 | 后续可扩展数据库、Excel、外部接口、规则库和模型能力 |
| 合规 | 不替代人工决策，不输出无证据事实，不绕过内部审批流程 |

## 11. 验收用例

| 用例 | 前置条件 | 操作 | 预期结果 |
|---|---|---|---|
| 资金流入关联方识别 | 已导入客户、贷款、流水、关联方数据 | 输入资金用途核查任务 | 输出资金用途风险、涉及金额、交易对手、证据编号 |
| 回款下降识别 | 已导入连续周期流水 | 输入回款下降分析任务 | 输出下降比例、对比周期、统计口径、证据编号 |
| 新增司法风险识别 | 已导入外部风险数据 | 输入外部风险扫描任务 | 输出司法风险事项、发生日期、来源和建议动作 |
| 字段缺失处理 | 流水数据缺少交易日期 | 发起回款分析 | 返回结构化错误，提示补充或映射交易日期字段 |
| 多轮追问 | 已完成一次分析 | 输入“按交易对手拆分” | 基于上一轮结果重新生成拆分结果和新版本 |
| 报告生成 | 已完成风险扫描 | 选择生成 PDF/Markdown | 报告包含数据来源、风险信号、证据链和复核事项 |

## 12. 业务待确认问题

1. 首期试点是否以公司贷款、流动资金贷款为主。
2. 放款后首次检查的默认时间范围是 7 天、15 天、30 天还是其他。
3. 回款下降的预警阈值采用固定比例还是机构自定义。
4. 关联方名单由系统导入、人工维护还是从外部工商关系识别。
5. 报告模板是否需要符合现行贷后检查报告格式。
6. PDF 报告是否需要电子签章、二维码或防篡改水印。
7. 数据归档期限和历史分析结果复用范围。
8. 是否允许接入外部大模型，是否必须本地化部署。
